# AI剪口播 — Design Context

> 本文件定义此技能的设计原则，指导所有 UI/UX 相关决策。

> **每次对本项目做任何改动后，必须同步更新 `CHANGELOG.md`，按 `## YYYY-MM-DD` 日期分组追加变更内容。**

## Users

- **主要用户**：视频内容创作者（口播类）
- **使用场景**：上传口播视频 → AI 预标口误和静音段 → 用户在网页审核确认 → 导出 FCPXML 给剪映 / FCP 完成最终剪辑
- **核心任务**：快速定位要删除的片段，一键导出剪辑工程
- **情感诉求**：标注过程应该是轻松愉悦的，好的工具让人感到顺手，而非枯燥乏味

## Brand Personality

- **调性关键词**：轻盈、克制、专业、愉悦
- **情感目标**：像好的效率工具一样顺手，偶尔有小小的愉悦感，不沉闷也不花哨
- **功能优先**：每个视觉元素都必须有功能目的，不做无意义的装饰

## Aesthetic Direction

- **视觉风格**：温暖极简 — 暖白底色 + 青色点缀 + 琥珀色辅助
- **配色体系**（亮色）：
  - 背景：`#F8F7F4`（暖白）
  - 表面：`#FFFFFF`
  - 主色调：`#0F766E`（青色）
  - 删除态：`#DC2626`（红色）
  - AI 推荐态：`#92400E`（琥珀色）
  - 中性文字：`#1A1917` → `#9C9A97`
- **配色体系**（暗色）：`prefers-color-scheme: dark` 自动切换，深炭底 + 青色主色调
- **字体**：Plus Jakarta Sans（400/500/600/700）
- **圆角**：适中（`10px` 大面，`6px` 按钮/小元素）
- **阴影**：轻量（`0 1px 3px`），不厚重

## Anti-References

- **不要**：赛博朋克、科技感、霓虹灯效果
- **不要**：紫色蓝色渐变、渐变文字、玻璃拟态
- **不要**：过度的暗色主题、发光边框
- **不要**：大阴影、厚重感、强对比的科技风
- **不要**：AI 生成的标配效果（glow borders, neon on dark, glass cards）

## Design Principles

1. **克制的愉悦感** — 添加恰到好处的微交互（按钮按压、加载动画、数字过渡），但绝不阻塞功能或分散注意力
2. **温暖的极简** — 暖色系而非冷灰，营造专业但友好的氛围；青色点缀传递专业感
3. **清晰的语义** — 颜色有明确含义（青色=操作/当前、红色=删除、琥珀色=AI推荐）
4. **无障碍优先** — WCAG AA 对比度、44px 最小点击区域、深浅主题自适应
5. **功能驱动** — 视觉设计服务于剪辑任务，让视频内容成为焦点，界面不应喧宾夺主

## 脚本参数规范

所有生成脚本**必须**接受输出目录作为参数，禁止使用相对路径写入 cwd：

| 脚本 | 用法 |
|------|------|
| `gen_analysis.js` | `node gen_analysis.js <subtitles.json> <输出目录>` |
| `generate_review.js` | `node generate_review.js <subtitles.json> <auto_selected.json> <audio.mp3> [输出目录]` |

HTTP Header 中禁止包含非 ASCII 字符，必须使用 RFC 5987 编码（`filename*=UTF-8''`）。

## 架构决策记录

### analysis.txt 格式

**格式简化为 `序号: 文本`（如 `0: 直接开始了啊`）。**
原因：原格式 `句N|idx X-Y|文本` 让 AI 同时处理"理解格式"和"识别口误"两件事，既慢又容易出错。简化后 AI 只看纯文本，idx 映射交给 `sentence_map.json` + `merge_selections.js`。

**`[静Xs]` 字段已删除。** 对口误识别无帮助，信息冗余。

**词级参考（`idx[N] 词 词 词`）已删除。** 整句删除不需要词级 idx，场景极少见，删除减少 context 消耗。

### AI 分析与合并流程

**AI 只输出句号数组（如 `[2, 3]`），合并由 `merge_selections.js` 执行。**
原因：(1) AI 不操作文件，只负责认知判断；(2) 句号比 idx 更直观，AI 处理更快更准；(3) 句号→idx 映射由 `sentence_map.json` 提供，代码自动展开。

**口误分析不使用 subagent。**
原因：analysis.txt 数据量小（数十句），无法并行，主 agent 等待 subagent 无时间收益，反而多一个故障点。

### 音频处理

**`detect_audio_profile.js` 及分贝阈值逻辑已全部删除。**
原因：该脚本跑 4 次 ffmpeg 检测音频分贝特征，生成 `audio_profile.json`，但后续流程未实际使用这些数据。删除后节省约 4 次 ffmpeg 开销和 1.5s 硬等待（`sleep 0.5 + sleep 1`）。
